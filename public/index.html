OHTTPd

HTTP/1.1 static file server

What the server does

OHTTPd serves static files from a root directory over HTTP. Point it at a folder and it will serve HTML, CSS, JavaScript, images, fonts, JSON, and anything else you put there. No configuration files, no scripting — just files on disk.

How it works

Send a request to the server's address. The server maps the URL path to a file in its root directory and sends it back with the correct Content-Type header. Directories are automatically resolved to their index.html file, if one exists.

$ curl -I http://localhost:8080/style.css
HTTP/1.1 200 OK
Content-Type: text/css; charset=utf-8
Content-Length: 148
Connection: close
Server: OHTTPd/1.0
